```
Build a full-stack content pipeline management app called "Traideas Content Pipeline" using Next.js (App Router) with a SQLite database. The entire app runs on a single port with API routes handled internally via Next.js route handlers. Use better-sqlite3 for the database (synchronous, no separate server needed). Style with Tailwind CSS and use lucide-react for all icons.

---

## TECH STACK

- Next.js 14+ (App Router)
- better-sqlite3 (SQLite, runs in-process, no separate DB server)
- Tailwind CSS
- lucide-react (icons throughout)
- No authentication — browser fingerprint (navigator.userAgent + screen dimensions stored in localStorage) is used to "pin" the current user per device

---

## DATABASE SCHEMA (initialize on first run via db/init.ts)

```sql
CREATE TABLE IF NOT EXISTS team_members (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  initials TEXT NOT NULL,
  created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS weekly_operators (
  week_key TEXT PRIMARY KEY,  -- ISO date of Sunday e.g. "2026-03-29"
  member_id TEXT NOT NULL,
  FOREIGN KEY (member_id) REFERENCES team_members(id)
);

CREATE TABLE IF NOT EXISTS contents (
  id TEXT PRIMARY KEY,
  member_id TEXT NOT NULL,
  week_key TEXT NOT NULL,       -- Sunday of that week ISO date
  title TEXT DEFAULT '',
  type TEXT DEFAULT '',         -- enum key
  mediums TEXT DEFAULT '[]',    -- JSON array of strings
  status TEXT DEFAULT 'pending_title',
  link_post TEXT DEFAULT '',
  link_doc TEXT DEFAULT '',
  link_file TEXT DEFAULT '',
  publish_date TEXT DEFAULT '',
  carried INTEGER DEFAULT 0,    -- 1 if carried from a previous week
  created_at TEXT DEFAULT (datetime('now')),
  updated_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (member_id) REFERENCES team_members(id)
);

CREATE TABLE IF NOT EXISTS feedback (
  id TEXT PRIMARY KEY,
  content_id TEXT NOT NULL,
  reviewer_name TEXT NOT NULL,
  note TEXT NOT NULL,
  created_at TEXT DEFAULT (datetime('now')),
  FOREIGN KEY (content_id) REFERENCES contents(id)
);

CREATE TABLE IF NOT EXISTS browser_pins (
  browser_key TEXT PRIMARY KEY,  -- stored in localStorage, generated once per browser
  member_id TEXT NOT NULL,
  FOREIGN KEY (member_id) REFERENCES team_members(id)
);
```

Seed 5 sample team members on first run if the table is empty.

## NEXT.JS API ROUTES (all under /api/)

All database access happens only in these route handlers. The frontend fetches from these.

- GET/POST   /api/members          — list all, add new
- DELETE     /api/members/[id]     — remove member
- GET/POST   /api/contents         — list by week_key query param, add new
- PATCH      /api/contents/[id]    — update title, type, mediums, status, links, publish_date
- POST       /api/feedback         — add feedback note { content_id, reviewer_name, note }
- GET        /api/operators        — get operator for a week_key
- POST       /api/operators        — set operator for a week { week_key, member_id }
- GET/POST   /api/pins             — get pinned member for a browser_key, set pin

---

## WEEK LOGIC (all in a utility file lib/weeks.ts)

- A "week" runs Sunday to Thursday (5 days shown in the calendar strip)
- week_key is always the ISO date string of that Sunday (e.g. "2026-03-29")
- When navigating to a week, auto-create a "pending_title" content slot for any member who has no content that week AND no content in "writing", "content_submitted", or "revision" status from any prior week (those are "carried" and shown as carried items this week)
- When a content is marked "scheduled" or "published", auto-create a new pending_title slot for that member in the NEXT week if one doesn't exist
- Default starting week is the current week's Sunday

## BROWSER PIN FEATURE

- On first load, if no browser_key in localStorage, generate one: `crypto.randomUUID()` and store as `localStorage.setItem('tcp_browser_key', key)`
- On load, fetch GET /api/pins?browser_key=KEY to get pinned member
- Store pinned member in React state (not localStorage — always fetched from DB)
- Top-right "Set your identity" button: opens a small modal/dropdown showing all team members as avatar + name options. Click to pin. Calls POST /api/pins with { browser_key, member_id }
- Once pinned, show the avatar + name in top right with a "Change" option
- The pinned user's name pre-fills the "reviewer name" field when adding feedback
- Each browser independently pins its own user — different browsers/devices can be different users

## ADD MEMBER FORM (inline panel)

Fields: Full name (text), Initials (text, max 2 chars, auto-derived from name if blank)
On save: POST /api/members, then auto-create a pending_title slot for this member for the current week

---

## ADD CONTENT FORM (inline panel)

Fields: Member (dropdown), Title (optional text input)
On save: POST /api/contents. If title provided, status = 'title_submitted', else 'pending_title'. Replace any existing pending_title slot for that member+week.

## MISC REQUIREMENTS

- All data fetching uses React's useEffect + fetch (no external data-fetching library needed)
- Use a single db singleton in lib/db.ts that initializes better-sqlite3 once and runs migrations/seed on first boot
- All IDs are generated with crypto.randomUUID() on the server side in route handlers
- week_key computation: given any date, find its preceding Sunday (getDay() offset), return ISO date string
- Operator auto-assignment: when navigating to a week with no operator set, randomly pick a member and POST to /api/operators
- Tailwind for all styling — no external component library needed except lucide-react
- Avatar circles: colored by member index mod 5 using Tailwind color classes (blue, green, yellow, red, purple)
- Mobile-friendly: the pipeline table and board should be horizontally scrollable on small screens
- All status badges use icon + label text, never icon alone
- The detail panel should smoothly scroll into view when opened
